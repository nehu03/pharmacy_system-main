Med it Easy — Supabase setup (after code changes in this folder)

1) Create a project at https://supabase.com

2) Project Settings → API: copy Project URL and anon public key into js/supabase-config.js
   (see js/supabase-config.example.js). The file is tracked in this repo for convenience.

3) SQL Editor: paste and run the full contents of:
   supabase/migrations/20250329120000_init_pharmacy.sql
   Success looks like: finished with no errors (DDL often shows "Success. No rows returned").

4) Authentication → URL configuration: add every URL where users open the app:
   - Local: http://127.0.0.1:8765 (or your static server port)
   - Production: https://YOUR-PROJECT.vercel.app (and custom domain if you add one)
   Put the primary URL in Site URL and the same list in Redirect URLs.

5) For local testing, Authentication → Providers → Email: consider turning OFF
   "Confirm email" so signUp returns a session immediately (needed for admin RPC flow).

6) Change the admin secret: edit claim_admin_role in the migration (or create a new migration)
   and keep login.html / registration in sync — do not ship the default secret to production.

--- Vercel (static deploy) ---

7) Import this GitHub repo in Vercel → New Project.

8) Root Directory — pick ONE approach (wrong setting causes 404 NOT_FOUND):
   A) RECOMMENDED: set Root Directory to exactly:   pharmacy_system-main
      (inner folder with login.html / js/ / index.html). Leave Build empty. Framework: Other.
      Your site is served at https://YOUR-PROJECT.vercel.app/ with no extra path.
   B) ALTERNATIVE: leave Root Directory EMPTY (repo root). Framework: Other.
      A vercel.json at the repo root rewrites requests into pharmacy_system-main/.

9) Framework: Other. Leave Build Command empty. Deploy.
   IMPORTANT: This app is static HTML (no Next.js, no package.json with "next").
   In Vercel: Project → Settings → General → Framework Preset = "Other" (not Next.js).
   If you see "No Next.js version detected" or "package.json has next", you picked Next.js
   or the wrong Root Directory. Fix Framework Preset + Root Directory, then Redeploy.

10) After deploy, copy your live URL (e.g. https://xxx.vercel.app) and add it in Supabase
    Authentication → URL configuration (step 4).

11) index.html at the app root redirects to login.html so opening / on Vercel works.
